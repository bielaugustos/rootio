## Implementação Técnica

### 1. Fluxo Completo
```
Login (qualquer método)
    ↓
Verificar ior_onboarding_done
    ↓
[Se não existir]
    → /onboarding (Step 1)
    ↓
Passar por todos os steps
    ↓
[Step 6 - Conclusão]
    → IR PRO HOJE → Home
    → ATIVAR NOTIFICAÇÕES → Permission request
    ↓
Salvar ior_onboarding_done=true
```

### 2. Flags no localStorage
- `ior_onboarding_done=true` - após completar onboarding
- `ior_onboarding_skipped=true` - se usuário pular
- `ior_onboarding_choices` - escolhas do usuário (JSON)
- `ior_first_habit` - primeiro hábito criado
- `ior_first_goal` - primeira meta financeira
- `ior_avatar` - avatar selecionado
- `ior_username` - nome do usuário

### 3. Nova Página de Onboarding
Criar `src/pages/Onboarding.jsx`:
- Estado `currentStep` (1-6)
- Estado `selectedOptions` (array de escolhas)
- Estado `firstHabit` (dados do hábito)
- Estado `firstGoal` (dados da meta)
- Estado `avatar` (avatar selecionado)
- Estado `username` (nome do usuário)
- Funções de navegação
- Integração com App.jsx

### 4. CSS
Criar `Onboarding.module.css`:
- Estilo neobrutalista (fundo --sun #FFD23F)
- Checkboxes personalizados (conforme tema)
- Animações entre steps
- Cards de seleção
- Avatar grid
- Cards bloqueados

### 5. Integração Login → Onboarding
Em `Login.jsx`:
```javascript
function handleSuccess() {
  // Verificar se é primeiro login ou usuário existente
  const hasExistingAccount = localStorage.getItem('ior_auth_skipped') === 'false' || session?.user
  
  if (hasExistingAccount) {
    // Usuário já fez login antes → ir direto para home
    navigate('/')
  } else {
    // Primeiro login → mostrar onboarding
    navigate('/onboarding')
  }
}
```

**Regra**: O onboarding sempre aparece no primeiro acesso. Se detectar que o usuário já criou uma conta anteriormente (tem sessão ativa), a sincronização é automática e o onboarding não aparece.

### 6. Notificações (Step 6)
Usar Notification API para solicitar permissão:
```javascript
async function requestNotificationPermission() {
  if ('Notification' in window) {
    await Notification.requestPermission()
  }
}
```

## Detalhes de Implementação

### Navigation Flow
- Step 1: COMEÇAR → Step 2
- Step 1: JÁ TENHO CONTA → Login (email)
- Step 2-5: VOLTAR (volta step anterior) / PULAR (próximo step)
- Step 6: IR PRO HOJE → Home / ATIVAR NOTIFICAÇÕES → Permission request

### Data Storage
- Após cada step, salvar progress no localStorage
- Ao completar Step 6, salvar `ior_onboarding_done=true`
